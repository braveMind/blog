hashSet细节

2016-06-15

JDK如下定义HashSet
This class implements the Set interface, backed by a hash table
(actually a HashMap instance). It makes no guarantees as to the iteration order of the set; in particular, it does not guarantee that the order will remain constant over time. This class permits the null element.

HashTable 是基于hash表的数据结构，实际上使用的是HashMap的数据结构,HashSet不保证结迭代顺序，也不保证保证表中的元素顺序随着时间的变化,HashSet允许包含一个null值。由于HashSet使用的是hashMap为底层的数据结构，那么hashSet也是线程非同步的。

HashSet add方法

hashSet实际上add元素时候是把将要存入的这个元素当做key值调用hashMap的put方法，放入hashMap中的，value放一个空的对象。因此hashSet,put元素的时间复杂度基本上是o(1),除了hash冲突。

hashSet contain方法

hashMap中没有contain放，hashMap中的方法叫做containKey曾经一道面试题。

没错，hashSet的contains方法是调用map.containsKey(O)方法的，containsKey(o)是根据hash函数去做散列的，所以与元素的多少无关，无论是多少元素containsKey(o)的时间复杂度基本上O(1)。

hashSet 的遍历 iterator

HashSet的遍历也是调用HashMap的keySet().iterator()进行遍历的Entry这样的一个链表节点的。

总结

HashSet可以看成是HashMap的一种特殊结构,HashSet存储的元素是仅仅是HashMap的key，因此hashSet可以过滤重复的值，同时采用了Hash表结构HashSet是一种非线程安全的存储结构，在多线程下需要线程同步。影响HashSet性能的因素和HashMap相同同样是初始化容量，和负载因子。

ArryList的细节

2016-06-13

ArrayList的定义

JDK如下定义:

所有的测试基于 jdk1.7
Resizable-array implementation of the List interface. Implements all optional list operations, and permits all elements, including null. In addition to implementing the List interface,this class provides methods to manipulate the size of the array that is used internally to store the list. (This class is roughly equivalent to Vector, except that it is unsynchronized.)

ArryList是一个可变数组，实现了List接口，可以存放所有的数据类型，例如null类型。ArrayList这个类在内部实现了控制数组长度的方法，用来存放list中的元素.
这个类除了是线程非同步的，其大致相当于Vector类。

ArrayList如何自动扩容

ArryList 如果不进行初始化数组大小，它的容量默认值是10，如果新增的元素长度小于默认数组的长度那么默认长度就是这个数组最小值，如果新增的长度大于数组的最大值，那就分配Integer范围内的最大值，如果大于这个最大容量就会报出 OutOfMemoryError这个异常，如果新增数组小比默认长度大又小于Interger的最大容量，那么增长为原本的1.5倍。

add元素比较简单，每次add之前先判断是否需要给数组扩展容量，然后数组 下标+1，把元素放入数组中。

总结：

（顺序插入元素ArrayList性能高）操作是在一列数据的后面添加数据而不是在前面或中间,并且需要随机地访问其中的元素时,使用ArrayList会提供比较好的性能；

（中间插入，删除LinkedList性能高，因为数组需要移动位置，链表只需要交换两个节点之间的指针）当你的操作是在一列数据的前面或中间添加或删除数据,并且按照顺序访问其中的元素时,就应该使用LinkedList了。
