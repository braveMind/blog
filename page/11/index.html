having 和 where 的区别

having vs where

子查询

子查询不能直接出现在where中，应该出现在 where的子查询里面。

in 和 not in 子查询

用来比较一个给定的值有没有出现在一个特定的集合里面。

Any some all

Any 与比较操作符连用，表示任何大于子查询的一项都会被返回。
some any的别名比较不常用。
all 与子查询比较，比子查询大的所有值，才会被返回。

查询性能优化

绝大多数的子查询都可以改成链接查询。

union 和 union all

必须有相同的列数目，并且列的数据类型必须相同。
union 不允许有重复的列，union 允许有重复的列。

数据库主外键对性能影响

1.数据库需要维护外键的内部管理；
2.外键等于把数据的一致性事务实现，全部交给数据库服务器完成；
3.有了外键，当做一些涉及外键字段的增，删，更新操作之后，需要触发相关操作去检查，而不得不消耗资源；
4.外键还会因为需要请求对其他表内部加锁而容易出现死锁情况；

数据库优化

索引加快了检索速度，降低了在索引里面插入，删除，以及修改的速度。因为写入操作不仅要写入到数据行，而且还有改变索引。其次索引要占用很大的磁盘空间。

添加的索引一般是给where子句添加。

sql的链接类型

内连接（inner join）

内连接是应用程序默认的链接类型，内连接是把表A和B的列组合在一起产生的，其组产生的值满足笛卡尔积.
inner join分为：

  1. 等值链接
  2. 不等值链接
  3. 自然链接

显示的使用 INNER JOIN

隐示的使用 INNER JOIN

外联接

  1. 左外链接
  2. 右外链接
  3. 全连接：完整外部联接返回左表和右表中的所有行。当某行在另一个表中没有匹配行时，则另一个表的选择列表列包含空值。如果表之间有匹配行，则整个结果集行包含基表的数据值

on和where的区别

1、 on条件是在生成临时表时使用的条件，它不管on中的条件是否为真，都会返回左边表中的记录。

2、where条件是在临时表生成好后，再对临时表进行过滤的条件。这时已经没有left join的含义（必须返回左边表的记录）了，条件不为真的就全部过滤掉。

left join 是left outer join的简写。

right join 是 right outer join 的简写。

linux系统中四种环境变量

2016-05-16

全局环境变量

/etc/profile

次文件是系统文件，配置在次文件的环境变量每个用户都可以使用,为了系统安全期间一般不建议使用次文件。

/etc/bashrc

次文件是每次 bash shell被打开时候该文件就会被读取里面的环境变量。

用户环境变量

~./bash_profile

每个用户都可使用该文件输入专用于自己使用的shell信息,当用户登录时,该文件仅仅执行一次!默认情况下,他设置一些环境变量,执行用户的.bashrc文件.

使用export 在终端导入

例如使用导入jdk环境变量：
export PATH=$PATH:/usr/local/lib/jdk1.6.0_25

修改环境变量文件生效方法

source 文件名

添加环境变量

linux 系统中环境变量的路径叫做 PATH,我们如果定义环境变量千万不要覆盖了系统PATH，一般的做法是使用 “：” 跟着我们自己的环境变量，和windows不同，windows使用“；”。

环境变量文件执行顺序

括弧里面的 环境变量文件名称可能 不同版本的linux 文件名字不一样,例如Ubuntu里面只有~/.profile

/etc/profile -> (~/.bash_profile | ~/.bash_login | ~/.profile) -> ~/.bashrc -> /etc/bashrc -> ~/.bash_logout

备忘录模式（行为模式）

2016-05-14

备忘录模式的定义：不破坏封装的前提下，捕获一个对象的内部状态，并在该对象之外保存这个状态，这样可以在以后将对象恢复到原先保存的状态。备忘录模式是为对象的保存和恢复设计的。

类的结构图

分析

备忘录模式是一种比较简单的把对象关键状态保存下来的模式，在需要用到这些状态的额时候进行还原。

备忘录模式的组成：

  1. 存储类（Memo），用来记录需要备忘的节点。
  2. 需要备忘的类（OriginalState）,程序中需要保存这个类的关键状态。
  3. 备忘录类，用来存储 Memo类，也就是对象的所有状态都会保存到这里。

优点

  1. 给用户提供了一种可以恢复状态的机制。可以是用户能够比较方便地回到某个历史的状态。
  2. 备忘录中的对象，可以起到对被保存对象的封装，从而可以恰当地保持封装的边界。

缺点

  1. 备忘录中添加过多的复杂的对象，会浪费很大的空间。
