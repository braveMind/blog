访问者模式（行为模式九）

2016-05-12

访问者模式是比较难理解的一种设计模式，封装一些作用于某种数据结构中的各元素操作，它可以在不改变数据结构的前提下定义作用于这些元素的新的操作。

类的结构图

分析

访问者模式的组成：

  1. 抽象的被访问对象（ComputerPart）,定义添加访问者
  2. 抽象的访问者对象（ComputerPartVisitor），用来传递被访问者对象
  3. 具体的被访问者对象就是普通的java类。
  4. 具体的访问者对象用来处理传递过来的被访问者对象。

优点

  • 好的扩展性
    能够在不修改对象结构中的元素的情况下，在访问者类中为对象结构添加新的功能。

  • 灵活性高
    可以通过访问者来定义整个对象结构通用的功能，从而提高复用程度。

  • 符合单一职责
    可以通过访问者来分离无关的行为，把相关的行为封装在一起，构成一个访问者，这样每一个访问者的功能都比较单一。 　　

    缺点

  • 具体元素对访问者公布细节。 
  • 具体元素变更比较困难，改变了具体元素就需要改变调用它的访问者。
  • 违背了依赖倒置原则，访问者依赖的是具体的元素，而不是抽象元素。依赖实现类而不是接口，扩展比较困难。

模板模式（行为模式八）

2016-05-11

模板模式把一系列不变的操作封装在模板这个抽象类中，把模板方法延迟在子类中去实现。

分析

组成：

  1. 顶层抽象类
  2. 子类实现模板方法。

模板类比较简单，在模板中规定了执行步骤，子类去实现。这些步骤就按一定的顺序执行。

优点

取点

模板父类中的方法依赖于子类中的方法，在复杂的项目中会带来代码阅读的难度。

策略模式（行为模式七）

2016-05-10

策略模式允许类根据不同的条件在运行的时候可以选择不同的策略（算法）处理问题。策略模式把一系列的算法包装起来，供Client根据不同的条件调用。

类的结构图

分析

结构组成：

  1. 顶层策略类
  2. 策略类的实现类
  3. 策略容器

策略模式的类结构比较简单，只要实现一个接口就可以实现一种策略，容器中含有对策略类的引用，客户端调用时候根据不同的条件，把不同的策略注入到容器中，就可以完成对这种策略的引用。

优点

策略模式封装的是算法，在算法不确定的时候，可以使用策略模式。如果算法改变时候，客户端可以不用做任何的调整。

缺点

策略模式在调用的时候，客户端必须知道这个具体使用哪个算法，不符合迪米特法则，可以使用工厂方法避免。

空对象模式（行为模式六）

2016-05-09

空对象模式用来代替在程序中检查null操作，它内置处理 null 对象的方法，增强了系统的稳定性。

类的结构图

分析

写程序中经常会遇到

客户端调用的时候必须先判断是否是空对象，如果忘记写空检测就会报 null point Exception。空对象模式就是解决这种问题的，它事先会对空值进行处理，这样客户端调用时候就会避免类似的空指针异常，客户端程序就比较优雅，同时系统稳定性提高。

组成：

  1. 顶层接口定义空对象检测方法。
  2. 具体子类实现接口，空对象实现和具体的实现。
  3. 对外提供工厂方法产生具体的对象。
  4. 工厂方法对外提供统一的对象生产方法。
