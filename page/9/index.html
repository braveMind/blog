过滤器模式(结构型模型三)

2016-04-23

过滤器模式比较单纯，就像漏斗一样，用来筛选满足条件的对象。可以筛选单一逻辑的对象，也可以使用逻辑运算And,OR把所筛选的单一条件链接起来。过滤器模式也属于结构型模型的一种。

类结构图

分析

定义高层过滤器接口，底层有不同的实现策略，产品类每次经过一个过滤器链就会被筛选一遍，得到最后的产品。

Main 方法

public class App {
    public static void main(String[] args) {
        Product product1=new Product("car","QQ");
        Product product2=new Product("Bus","大巴车");
        Product product3=new Product("Bicycle","小山地车");
        Product product4=new Product("car","奥拓");
        Product product5=new Product("Bus","超大巴车");
        Product product6=new Product("Bicycle","山地车");
        List<Product> list=new ArrayList<Product>();
        list.add(product1);
        list.add(product2);
        list.add(product3);
        list.add(product4);
        list.add(product5);
        list.add(product6);
        Filter bicycleFilter=new BicycleFilter();
        Filter busFilter=new BicycleFilter();
        Filter carFilter=new CarFilter();
        Filter andFilter=new AndFilter(carFilter,busFilter);
        Filter orFilter=new ORFilter(carFilter,busFilter);
        bicycleFilter.getByType(list);//获取 car 集合
        busFilter.getByType(list);//获取 bus 集合
        carFilter.getByType(list);//获取小汽车集合
        andFilter.getByType(list);//获取 既是 小车中的 大巴车,实际生活中没有这种车
        orFilter.getByType(list);//获取 小车 和 大巴车集合
    }
}

优点

过滤器类结构清晰简洁，可以方便的筛选符合类型的产品，而不关心产品的细节

缺点

对于同一类的产品可以方便的扩展，对于不同类的产品如果使用 Filter 就需要重写Filter类，不同产品不易于扩展。

尼采：在世纪的转折点上

2016-04-21

第一次接触尼采就是这本启蒙书，作者描述的尼采是那样的纯粹，那么的孤独。俯拾尼采遗落的语句让我心灵为之颤栗。也许是第一次真正接触西方哲学。

同时周国平的这本《尼采：在世纪的转折点上》让读者能从整体上对尼采有个印象，同时讲述了各个时期的尼采作品的创作背景。对于理解原作有很好的帮助。

原书摘录

  1. 耶稣说：把凯撒的给凯撒，把上帝的给上帝。

  2. 生物性和精神性是人性的两端，一个人唯有两端皆发育良好拥有健康的本能和高贵的灵魂，才是优秀的人。一个社会唯有为产生这样的人提供适宜的环境，由这些人组成，才是美好的社会。可是，我们的传统中，本能和精神都受到压制，我们宁肯牺牲两端以成全中间的社会性，结果得到的是一种抹杀个人的平庸的社会性。

  3. 尼采的哲学只确定两点：第一要有健全的生命本能，第二要有超越的精神追求。

  4. 大自然的星空，群星灿烂。那最早闪现的，未必是最亮的星宿。有的星宿孤独的燃烧着，熄灭了，很久很久以后，它的光才到达我们的眼睛。

  5. 谁终将声震人间，必长久深自缄默；谁终将点燃闪电，必长久如云漂泊。尼采的诗，作者翻译的真好。

桥接模式（结构型模型二）

2016-04-19

桥接模式：将抽象部分与它的实现部分分离，使它们都可以独立地变化。怎么理解呢？桥接模式解决的问题是当类有多个维度发生改变时候，把发生变化的因素写剥离到接口里，在抽象类中定义抽象方法这个方法是类中共有的，具体的实现类继承这个抽象类，实现抽象方法。客户端调用抽象类，传递接口的实现类，完成具体的方法调用。

类图

分析

桥接模式就是对大的接口进行拆分，把变化的，不可确定的接口剥离出来，然后在抽象类中进行组装，在抽象类的子类中实现父类中的方法。

优点

接口的实现类对客户端调用完全透明。接口实现类发生修改，客户端完全不关心，低耦合，拿这个例子来说，当手机颜色发生变化时候完，抽象类和客户端调用完全不用修改代码,符合开闭原则。

不足

那个类应该设计成接口，那个方法应该设计成抽象方法需要设计者考虑。

适配器模式（结构型模型一）

2016-04-18

结构模型官方的解释是：怎么用对象和类去构建一个更大的更灵活的结构。我的理解是，使用类和结构之间的结构组合对外提供一种更灵活的服务.

适配器模式：已经有的接口不能满足客户端的调用，需要引入适配器类来适配现有的接口对外提供服务，客户端调用适配器类就可以满足需求，而不需要更改已存在的类，使客户端和具体的接口类解耦。

优点

适配器可以把内部的接口进行重组，在不改写类源码的情况下重新对外提供一个统一对外的接口，提高了系统的灵活性。

不足

目标系统和调用系统之间存在一个代理类，在复杂的系统中代理类可能要做很多其它的服务，可能会导致性能损失和难以维护。
