简单工厂模式（创建型模型一）

2016-04-10

简单工厂模式属于设计模式中的创建型模型，解决的问题是隐藏对象的创建细节，根据参数工厂创建不同的对象。

说明：Bicycle和Car都属于交通工具，实现Toos接口，工厂类创建一个静态的方法create();根据不不同的参数来决定生产具体的产品，App负责调用。

示例代码

模式优点

  1. 将对象的创建和业务代码分离，降低系统耦合度，两者修改起来比较方便。
  2. 静态工厂使用起来简单，只需要传入一个正确的参数就能创建对象，实际开发中这个参数来可以来自于配置文件。

模式缺点

  1. 工厂类在系统中占得比重变大，一但工厂类需要修改或者逻辑变动整个系统都会受影响，不符合开闭原则。
  2. 不利于扩展，增加新的产品需要在工厂类中增加逻辑代码
  3. 有人说静态方法无法在子类中继承，不过这点我认为这是静态方法本身的局限性，属于语法层面的，放在这里有点牵强。

适用场景

  1. 系统中创建的对象种类不多，并且创建逻辑简单
  2. 需要对调用端隐藏创建细节，只需要提供给调用端一个参数，就能创建对象。

什么是设计模式？设计模式是软件设计过程中的最佳实践的总结由Erich Gamma, Richard Helm, Ralph Johnson and John Vlissides这四位前辈在1994年提出。

设计模式分类

设计模式分为以下四类：创建型模型，结构模型，行为模型，J2EE Patterns
参考

设计模式遵守的原则

设计模式遵守的原则有：单一职责原则， 里氏替换原则，依赖倒置原则，接口隔离原则，迪米特法则，开闭原则
参考：设计模式之禅-秦小波

思考

个人理解设计模式的引入，增加了系统更高层次的抽象，最主要的目的就降低了系统之间的耦合度。
设计模式让软件有了一套固定的模式对于代码的可维护性来说就大大增强了。但有也抵制设计模式，过于臃肿每个人的立场不同但是优秀的开发者总是能不自觉的找到一种好的程序组织模式，即使在使用函数式编程时候。所以不要为了使用设计模式而使用设计模式，设计模式解决的是一般常规性问题。

构造函数和原型

2016-04-10

什么是构造函数？ js中并没有区分什么是构造函数，构造函数本身和一般的函数没有什么区别，只是被当做对象的构造者，使用new关键字来调用，一般函数名首字母大写。

使用new关键字创建对象步骤

当构造函数被调用时候具体发生了什么？

  1. 创建了一个新的对象 test1
  2. 把test1的constructor指向了 Test
  3. 扩大了test1原型链，使是test1共享Test这个 构造函数的原型对象（注意是构造函数的原型,test1可以继承Test原型对象上的属性）
  4. 调用 Test()这个构造函数
解释：

1, 创建一个新对象，这个很好理解,这个test1是新创建的一个对象。

2, test1的构造函数为Test

3，为什么会扩大原型连查找？js中属性查找先会在对象本身，test1上进行查找，如果没有这个属性会去这个构造函数的原型对象上去查找，所以所有子对象都会共享这个函数的原型对象。

构造函数的其它trick

证明一个对象只有在new的时候才能属于某各类，中途中是无法手动的去改变这个类的所属的。但并不是说一个对象只能属于一个类，在javascript中又运行多继承，一个对象可以属于多个类。

系统更新后nginx报错

2016-04-09

nginx

/usr/local/var/run/nginx.pid” failed (13: Permission denied)

MAC 系统升级为OS X EI 后文件系统 /usr 用户目录没有权限导致nginx启动报错。原本以为是nginx.conf配置问题。使用 sudo 启动可以启动，但是通过http代理的时候访问代理路径 http报403而且并不是代理路径没有权限，而是nginx启动后需要写日志，需要往磁盘写pid所以导致403问题。

解决方法：

sudo chown -R 'username' run  #写pid的磁盘路径
chmod -R 700 run/

同理修改日志，log的目录权限最后问题解决。

linux防火墙开端口

2016-04-09

Linux环境下，常常需要自己开一些端口，基本上每次用的时候都要去找，所以就总结记录下来开端口常用的命令，下次直接可以使使用，拯救我懒惰的大脑。

linux 防火墙开端口

保存修改：

/etc/rc.d/init.d/iptables save

查看打开的端口：

/etc/init.d/iptables status

关闭防火墙

/etc/init.d/iptables stop
service iptables stop # 停止服务

查看防火墙信息

/etc/init.d/iptables status 

开放端口:8080

/sbin/iptables -I INPUT -p tcp --dport 8080 -j ACCEPT
/etc/init.d/iptables restart

将更改进行保存

/etc/rc.d/init.d/iptables save

另外也可以直接在/etc/sysconfig/iptables中增加一行：

-A RH-Firewall-1-INPUT -m state –state NEW -m tcp -p tcp –dport 8080 -j ACCEPT
