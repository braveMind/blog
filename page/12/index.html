策略模式（行为模式七）

2016-05-10

策略模式允许类根据不同的条件在运行的时候可以选择不同的策略（算法）处理问题。策略模式把一系列的算法包装起来，供Client根据不同的条件调用。

类的结构图

分析

结构组成：

  1. 顶层策略类
  2. 策略类的实现类
  3. 策略容器

策略模式的类结构比较简单，只要实现一个接口就可以实现一种策略，容器中含有对策略类的引用，客户端调用时候根据不同的条件，把不同的策略注入到容器中，就可以完成对这种策略的引用。

优点

策略模式封装的是算法，在算法不确定的时候，可以使用策略模式。如果算法改变时候，客户端可以不用做任何的调整。

缺点

策略模式在调用的时候，客户端必须知道这个具体使用哪个算法，不符合迪米特法则，可以使用工厂方法避免。

空对象模式（行为模式六）

2016-05-09

空对象模式用来代替在程序中检查null操作，它内置处理 null 对象的方法，增强了系统的稳定性。

类的结构图

分析

写程序中经常会遇到

客户端调用的时候必须先判断是否是空对象，如果忘记写空检测就会报 null point Exception。空对象模式就是解决这种问题的，它事先会对空值进行处理，这样客户端调用时候就会避免类似的空指针异常，客户端程序就比较优雅，同时系统稳定性提高。

组成：

  1. 顶层接口定义空对象检测方法。
  2. 具体子类实现接口，空对象实现和具体的实现。
  3. 对外提供工厂方法产生具体的对象。
  4. 工厂方法对外提供统一的对象生产方法。

    redis（挖坑）

    2016-05-08

    掌握

    1. Redis支持五种数据类型。
    2. Redis部分命令只能对特定的数据进行执行（Append只能针对字符串进行操作，HSET 只能对哈希表进行操作）而DEL，TYPE,EXPIRE可以对所有数据进行操作。
    3. Redis的数据库如何实现存储不同的数据类型的。
    4. 发布订阅，脚本，事务等特性。
    5. Redis如何处理客户端的请求。
    6. Redis位操作，排序，复制，Sentinel,集群。
    7. Redis单机特性，事务、持久化、Lua脚本、排序、二进制位操作。

    数据结构

    1. Redis是由键值对组成的，其中数据库的键值必须是字符串
    2. Redis的值value，可以是字符串，列表对象（）list object）,哈希对象（hash object）,集合对象（set object）,有序集合对象（sorted set object）这五种。
    3. Redis SDS 磁盘空间分配两种策略如果小于1M,那么分配free为字符串本身大小，如果大于1M那么free为1M，通过预分配策略可以减少内存分配的次数。

    状态模式（行为模式五）

    2016-05-08

    状态模式：状态模式又称为类的状态模式，描述的是一个类根据不同的状态处理不同的业务逻辑，通常我们在项目中会通过 if(state) else,或switch来处理，但是对于复杂的类来说这样的代码显然可读性差，而且耦合性高。使用状态模式可以把每个类对应的不同状态交给具体的类处理接口来处理。

    类图

    分析

    状态模式组成：

    1. 抽象状态(StateHandle） 处理接口。
    2. 具体状态(ConcreteState)处理接口。
    3. 具体类(Content)。

    具体类处理者对细节的处理交给抽象状态处理者的实现，替代if-else 逻辑耦合，客户端只需要调用具体类的 hand方法就可以，屏蔽状态处理的细节。

    代码

    顶层抽象类

    抽象类的实现

    public class ConcreteState implements StateHandle {
    @Override
    public void doAction(String state) {
        System.out.println("this request to concrete StateHandle "+state);
    }
}

    状态类 

    public class Content {
    private StateHandle state;
    private String stateFlag;

    public String getStateFlag() {
        return stateFlag;
    }

    public void setStateFlag(String stateFlag) {
        this.stateFlag = stateFlag;
    }

    public StateHandle getState() {
        return state;
    }

    public void setState(StateHandle state) {
        this.state = state;
    }
    public void HandleState(String stateFlag){
        state.doAction(stateFlag);
    }

}

    APP类调用 

    public class App {
    public static void main(String[] args) {
        StateHandle stateHandle=new ConcreteState();
        Content content=new Content();
        content.setState(stateHandle);//注入具体的处理类
        content.setStateFlag("Ready");
        content.HandleState(content.getStateFlag());//调用具体处理类

    }
}

    优点

    状态模式相对于 if -else只是把 if-else 对应的代码交给具体的状态处理着，避免了写一堆逻辑代码，代码结构清晰。解耦了逻辑处理与业务判断的耦合性，方便扩展。
