系统更新后nginx报错

2016-04-09

nginx

/usr/local/var/run/nginx.pid” failed (13: Permission denied)

MAC 系统升级为OS X EI 后文件系统 /usr 用户目录没有权限导致nginx启动报错。原本以为是nginx.conf配置问题。使用 sudo 启动可以启动，但是通过http代理的时候访问代理路径 http报403而且并不是代理路径没有权限，而是nginx启动后需要写日志，需要往磁盘写pid所以导致403问题。

解决方法：

sudo chown -R 'username' run  #写pid的磁盘路径
chmod -R 700 run/

同理修改日志，log的目录权限最后问题解决。

linux防火墙开端口

2016-04-09

Linux环境下，常常需要自己开一些端口，基本上每次用的时候都要去找，所以就总结记录下来开端口常用的命令，下次直接可以使使用，拯救我懒惰的大脑。

linux 防火墙开端口

保存修改：

/etc/rc.d/init.d/iptables save

查看打开的端口：

/etc/init.d/iptables status

关闭防火墙

/etc/init.d/iptables stop
service iptables stop # 停止服务

查看防火墙信息

/etc/init.d/iptables status 

开放端口:8080

/sbin/iptables -I INPUT -p tcp --dport 8080 -j ACCEPT
/etc/init.d/iptables restart

将更改进行保存

/etc/rc.d/init.d/iptables save

另外也可以直接在/etc/sysconfig/iptables中增加一行：

-A RH-Firewall-1-INPUT -m state –state NEW -m tcp -p tcp –dport 8080 -j ACCEPT
