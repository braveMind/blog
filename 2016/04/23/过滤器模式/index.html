过滤器模式(结构型模型三)

2016-04-23
  1. 1. 类结构图

过滤器模式比较单纯，就像漏斗一样，用来筛选满足条件的对象。可以筛选单一逻辑的对象，也可以使用逻辑运算And,OR把所筛选的单一条件链接起来。过滤器模式也属于结构型模型的一种。

类结构图

分析

定义高层过滤器接口，底层有不同的实现策略，产品类每次经过一个过滤器链就会被筛选一遍，得到最后的产品。

public class App {
    public static void main(String[] args) {
        Product product1=new Product("car","QQ");
        Product product2=new Product("Bus","大巴车");
        Product product3=new Product("Bicycle","小山地车");
        Product product4=new Product("car","奥拓");
        Product product5=new Product("Bus","超大巴车");
        Product product6=new Product("Bicycle","山地车");
        List<Product> list=new ArrayList<Product>();
        list.add(product1);
        list.add(product2);
        list.add(product3);
        list.add(product4);
        list.add(product5);
        list.add(product6);
        Filter bicycleFilter=new BicycleFilter();
        Filter busFilter=new BicycleFilter();
        Filter carFilter=new CarFilter();
        Filter andFilter=new AndFilter(carFilter,busFilter);
        Filter orFilter=new ORFilter(carFilter,busFilter);
        bicycleFilter.getByType(list);//获取 car 集合
        busFilter.getByType(list);//获取 bus 集合
        carFilter.getByType(list);//获取小汽车集合
        andFilter.getByType(list);//获取 既是 小车中的 大巴车,实际生活中没有这种车
        orFilter.getByType(list);//获取 小车 和 大巴车集合
    }
}

优点

过滤器类结构清晰简洁，可以方便的筛选符合类型的产品，而不关心产品的细节

缺点

对于同一类的产品可以方便的扩展，对于不同类的产品如果使用 Filter 就需要重写Filter类，不同产品不易于扩展。
