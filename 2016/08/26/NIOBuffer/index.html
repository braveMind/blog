Buffer存放一大块的数据，这些数据可以被写入channel中或从channel中读入到缓冲区中，buffer是一个对象也充当着NIO的一个端点。Buffer提供了一种访问数据的机制，并可以跟踪读写处理机制。

Buffer有别于原始的阻塞I/O,先前的字节流读写是单向的，现在可以从Buffer里面读数据，也可以写数据到buffer里面，在NIO里面Channels是streams的同意词。

NIO Buffer的特点。

  • Buffer是构建NIO的基础
  • Buffer是定长的读写容器
  • 每一个Buffer都是可读的，但不是所有的buffers都是可写的。
  • Buffer是Channels的终端。
  • 在只读缓冲区的内容是不可变的，它的标志、位置和范围是可变的
  • 默认的情况下，Buffer不是线程安全的。

Buffer 的类型

每一个原始类型都有一个Buffer类型与之对应，所有的类型都实现了Buffer接口，使用最频繁的是ByteBuffer,下面是java NIO 包下的buffer类型。

  • ByteBuffer
  • CharBuffer
  • ShortBuffer
  • IntBuffer
  • LongBuffer
  • FloatBuffer
  • DoubleBuffer
  • MappedByteBuffer

Buffer Capacity

Buffer是固定长度的类型，我们可以存储大量的定长数据，buffer的最大固定长度叫做Buffer的容量。一旦Buffer满了，就需要在写之前情况buffer的容量，一旦buffer的长度固定那么这个值就不会改变了。

Buffer Limit

在写的模式下，limit最大等于capacity，在读的模式下，limit等于最后以读到数据的最后一位，当数据开始写的时候limit开始增长，Limit始终大于O，小于等于capacity.0 <= limit <= capacity

Buffer Position

Position指向当前buffer的中的当前位置，当buffer被创建的时候position指向0,在读或者写的时候当前位置开始增加，指向下一个位置，position总之介于0和limit之间。

Buffer Mark

Mark是在buffer中设置一个书签，当mark()被调用，当前的缓冲区中position会被记录，当reset()被调用时候被标记的位置会被跟新。

Buffer flip,clear and rewind

Buffer flip()

flip（）方法是被用来进行 get操作，或着通知当前缓冲区需要一个连续的存储空间进行写操作。flip() 将缓存字节数组的指针设置为数组的开始序列即数组下标0。这样就可以从buffer开头，对该buffer进行遍历（读取）了.也就是说调用flip之后，读写指针指到Buffer头部，并且设置了最多只能读出之前写入的数据长度(而不是整个缓存的容量大小)。

Buffer clear()
clear()方法被用来执行缓冲区的put操作，或者让缓冲区准备好read操作，clear操作设置limit的指针指向缓冲区的最后位置，也就是指向capacity当前position指向0，开始准备读操作。

Buffer rewind()
rewind()方法被用来多次读取已经存在缓冲区的数据，rewind()方法设置缓冲区的当前位置为0；

如何从 NIO的Buffer中读取数据。

1，使用allocate(size)开辟固定大小的缓存区域。

ByteBuffer byteBuffer = ByteBuffer.allocate(512);

2,Flip操作后开始读取。

byteBuffer.flip();

3，可以被读入的字节数。

int numberOfBytes = fileChannel.read(byteBuffer);

4，从缓冲区中读取字符。

char c = (char)byteBuffer.get();

如何进行写入缓冲区

1，使用allocate(size)开辟固定大小的缓存区域。

ByteBuffer byteBuffer = ByteBuffer.allocate(512);

2，把数据放入缓冲区

byteBuffer.put((byte) 0xff);
