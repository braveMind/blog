物理机器内存模型

硬件的高速缓存很好的解决了处理器和内存的矛盾，但是引入了新的问题，缓存一致性，当多个处理器同时共享同一个主存时候可能导致数据不一致，解决这些问题，需要各个处理器访问缓存时遵循一些协议，在读写时候要根据协议来读写，这些协议有：MSI、MESI、Synpse、Firefly、Dragon Protocol等。

为了使处理器内部的运算单元被充分的利用，处理器可能会对输入的代码进行乱排序执行（Out-Of-Order Execution）优化，计算机会在处理之后将乱序的结果重组保证该结果与顺序执行的结果是一致的。但是并不保证在程序中各个语句的计算先后顺序与输入代码中的顺序是一致的，如果一个计算任务依赖另外一个计算机任务的中间结果那么这个中间结果是不可靠的，java虚拟机在即时编译的时候也有类似的指令重排序（Instruction Reorder）优化

java虚拟机内存

java内存模型主要定义各个变量的的访问规则，虚拟机中将变量存储到内存和从内存中取出变量的底层细节。这些变量指的是实例字段，静态字段，构成数组对象的元素，不包括局部变量与方法参数（线程私有变量）不会被共享。

每条线程有自己的工作内存，保存该线程使用的变量的主内存副本的拷贝，线程对所有变量的读取操作必须都在工作内存中进行，而不能直接读取主存中的的内存变量。不同线程之间无法直接访问对方工作内存中的变量，线程间变量的访问需要通过主内存来完成。

内存划分层次以及对应关系

这里所划分的内存模型和通常将的java虚拟机内存模型（heap,stack,method areas）的划分层次是不一样的,如果两者要勉强对应起来那么，主内存对应java中对象的实例数据部分，工作内存对应于虚拟机中栈中的部分区域。从更底层的层次来说主内存就是硬件的内存，为了获取更高的运行速度，虚拟机及硬件系统可能让工作内存优先存储于寄存器和高速缓存中。

内存间的交互操作

主内存与工作内存之间的交互协议，一个变量如何从主存copy到工作内存，如何从工作内存同步到主存，java定义了8种操作来完成。

  • lock(锁定)：作用于主内存变量，它把一个变量表示为一条线程独占状态。
  • unlock(解锁)：作用于主内存变量，它把一个处于锁定状态的变量释放出来，释放后的变量才可以被其它线程锁定。
  • read(读取)：作用于主内存变量，把变量从主内存传输到线程工作内存中，以便随后的load动作使用。
  • use(使用)：作用于工作内存变量，把工作内存中的变量传递给执行引擎，每当虚拟机遇到一个需要使用变量的值的字节码指令将会执行这个操作。
  • assign（赋值）：作用于工作内存，它把从一个执行引擎接收到的值赋值给工作内存变量，每当虚拟机遇到一个变量赋值的字节码指令的时候执行这个操作。
  • store(存储)：作用于工作内存变量，把工作内存中的变量传递到主内存中，一遍随后的write操作使用。
  • write(写入)：作用于主存中的变量，它把sotre操作从工作内存中得到的变量放入到主存变量中。

如果把一个变量复制到工作内存，那就顺序的执行read和load操作，如果要把变量从工作内存同步回主存，就要按顺序执行store和write操作。java内存模型要求上述操作是按照顺序执行的，而并没有保证是连续执行的 也就是说read和load之间，store和write之间可以插入其它的指令，如果对主存中的变量a,b 进行访问的时候一种出现的可能顺序是 read a、 read b、load b、 load a。

volatile 类型变量

关键字volatile变量提供了轻量级的同步机制，当一个关键字被定义为volatile，当一条线程修改这个变量的值，新的值对所有的线程都是立即可知的，普通变量做不到这一点，变量的值在线程间的传递均需要通过主存来完成。

基于volatile变量运算在并发下是安全的？

如果是非原子操作，volatile变量在并发运算中一样是不安全的。例如i++这样的运算，在各个线程取到i的时候i的值在各个线程中是一致的，在执行 add指令的时候其它线程可能把i的值已经修改了，在同步到主存的时候就可能把较小的值写到主存中。

由于volatile变量只能保证可见性，在不符合面的场景中依然需要（synchronized或java.util.concurrent中的原子类）保证原子性。

  • 运算结果并不依赖变量的当前值，或能够确保只有单一的线程修改变量的值
  • 变量不需要与其它变量共同参与不变的约束。

原子性、可见性、有序性

  • 原子性：java内存模型来保证原子性变量的操作包括read、load、assign、use、store、write。如果应用需要大范围的原子操作使用同步代码块，synchronized关键字因此使用synchronized之间的代码块也具有原子性。

  • 可见性：可见性就是当一个线程修改了共享变量的值，其它线程能够立即得知变量的修改。java内存模型是通过变量在修改后讲新值同步到主存，在变量读取前从主存刷新变量的值，主存作为传递媒介来实现变量的可见性，无论是volatile变量还是普通变量，普通变量不能保证每次变量改变时，这个值对别的线程可见。

    除了volatile变量，final,synchronized也可以实现可见性

  • 有续性： 如果在本线程内观察所有的操作都是有序的（不管怎么重排序，（单线程）程序的执行结果不能被改变。**），如果在另外一个线程观察所有的操作都是无序的。

前半句指的是线程内表现为串行，后半句指的是重排序和工作内存与主存同步延迟的现象。
