#Spring中的AOP的实现方式。

java中每一个思想的提出都是经历过很多痛苦的编程实践，AOP面向面向切面编程从切点中分离出来横切关注点，横切关注点就是与具体的业务无关的相关代码，可能是一些公用的模块，使原来分散在不同业务模块的代码可以统一的管理。

类的实现方式

Advice是AOP联盟定义的模型（org.aopalliance.aop）定义的空接口，Spring也是对AOP的一种实现。BeforeAdvice(package org.springframework.aop)定义空接口，MethodBeforeAdvice(package org.springframework.aop)交个用户自己实现。

用户实现

MethodBeforeAdvice

Method参数，是目标方法的放射对象，Object[] var2是目标方法的参数，Object var3目标对象。

<bean id="xxxService" class="org.springframework.aop.framework.ProxyFactoryBean">  
        <property name="interceptorNames">  
            <list>

        </property>  
    </bean>

使用注解方式提供

使用注解方式提供给需要的方法进行注解。

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyPermission {
    String role() default "";
}

@Component
@Aspect
public class MyPermissionImp {
        @Around(value= "@annotation(com.mvc.annotation.MyPermission)")
        private Object test(ProceedingJoinPoint joinPoint) throws Throwable {
            MethodSignature ms=(MethodSignature) joinPoint.getSignature();
            Method method=ms.getMethod();
            System.out.println(method.getAnnotation(MyPermission.class).role().toString()+"标记");
            Object object=joinPoint.proceed();
            return object;

        }

}

@Around 和 before 和 after 最主要的区别就是，Around方式可以决定目标方法是否需要执行！
