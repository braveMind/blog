Java NIO Channel

2016-08-26

在java NIO中，channel被用来进行I/O传输，Channel像是一个管道被用来从Buffer到另外一个（Entity）实体之间进行进行数据传输。Channel从Entity中读数据把它放入到缓存区中进行处理,同样的Channel可以写数据到buffer中，数据会被传递到channel的另外一端。

Channel是Java NIO提供和native I/O进行数据传输的一种方法。我们应该使用缓冲区和Channel和进行通信，因此Channel像是一个桥梁连接I/O操作的两端。Channel是用Buffer来进行数据的读写的。

Channel的特性

  1. 不像是stream,channels允许同时读写。
  2. Channel从缓冲区中读取数据,并从缓冲区中写入数据。
  3. Channel可以异步的读取数据和写入数据。
  4. Channel有两种模式一种是阻塞模式，另外一种是非阻塞模式。
  5. 非阻塞模式不会让线程处于休眠状态。
  6. 面向字节流的channel,像sockets只能使用sokets的非阻塞模型。
  7. 数据可以被从另外一个通道传递给另外一个通道，如果其中任何一个是FileChannel.

Java NIO Channel 类型。

如下的两种类型是java中常见的 Channel实现类。

FileChannel

基于文件读写的Chanel是不能被设置成非阻塞模式。

SocketChannel

有三种类型的Channel,SocketChannel,ServerSocketChannel 和 DatagramChannel
这些都是可以被选择的，工作在非阻塞I/O下。

Java NIO Channel 例子

``` java
package com.javapapers.java.nio;

import java.io.IOException;
import java.io.RandomAccessFile;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;

public class ChannelExample {
public static void main(String args[]) throws IOException {
RandomAccessFile file = new RandomAccessFile(“temp.txt”, “r”);
FileChannel fileChannel = file.getChannel();
ByteBuffer byteBuffer = ByteBuffer.allocate(512);
while (fileChannel.read(byteBuffer) > 0) {
// flip the buffer to prepare for get operation
byteBuffer.flip();
while (byteBuffer.hasRemaining()) {
System.out.print((char) byteBuffer.get());
}
// clear the buffer ready for next sequence of read
byteBuffer.clear();
}
file.close();
}
}
``

JAVA NIO 指南

我们开始学习JAVA NIO教程，这是第一篇文章因此我们当做目录。java的非阻塞IO操作是面向buffer的，以前的标准IO操作是面向字节流的。

标准IO和NIO的不同

第一个主要的不同点是，标准IO操作是基于字节流（stream）的，NIO是面向Buffer(缓存区的)，基于Buffer的操作处理数据更灵活。基于Buffer的IO操作只要数据到达缓冲区，那么这个数据就可以被处理。因此我们可以把数据指针向前或者向后移动，而字节流却不可以这么操作。

第二个主要的不同点是，IO操作，在基于字节流的情况下，线程会被阻塞知道完成IO操作。当NIO操作的时候线程不会被阻塞，如果IO操作未完成线程可以去做其它的事情而不必停留在阻塞模式下。NIO结合channels(通道)和selectors(选择器)一个多线程可以管理多个线程进行并行的IO操作。

CopyOnWriteArrayList

2016-08-24

JDK1.5以后的并发包中新增的几个容器，平时在码代码的时候压根都没有印象❤️,在多线程读取List的时候CopyOnWriteArrayList 是个不错的

为啥取个这么长的名字

CopyOnWriteArrayList，为什么会起个这么长的名字，对于英语不好的人来说多记忆一个单词就会死一个脑细胞，从字面上来讲 “复制(copy)发生（on）在写（write,其实这个写代表的是所有引起数组变化的操作）的ArrayList上”😒，不符合语义，应该把主语提前，再添加一些修饰语,”ArrayList每次发生写操作时候就重新复制一个数组出来”。对，这样合理多了，不过这只是对这个单词的第一印象。

jdk1.7官方发言

A thread-safe variant of {@link java.util.ArrayList} in which all mutative operations (add, set, and so on) are implemented by making a fresh copy of the underlying array.

private volatile transient Object[] array;

是ArrayList线程安全的变体所有的改变数组的操作都需要新建一个数组的的拷贝，并将新的值放在这个数组中，在把用 volatile 修饰的数组指向这个新的数组，如果不懂volatile关键字的用法请看《深入理解java虚拟机》第五章。

add 元素

add操作使用的是可重入锁，理论上更高效，但是程序员要不要使用lock锁存在很大的争议。
synchronized vs Lockhttp://blog.csdn.net/fw0124/article/details/6672522

集合遍历

我们如果多个线程在迭代器中去修改ArrayList会报
ConcurrentModificationException，如果尝试修改CopyOnWriteArrayList 会报一个 UnsupportedOperationException,是的这个异常是迭代器自定义的异常，人家根本就不支持在迭代时候去修改数组的结构！！ 好自为之吧，别遍历的时候修改数组的结构了。

CopyOnWriteArrayList使用的场景。

通过volatile提供了“读取到的数据总是最新的”这个机制的,多线程高效的遍历数组可以使用CopyOnWriteArrayList.

对于经常被修改的数据列表，使用同步的ArrayList性能胜过CopyOnWriteArrayList,因为CopyOnWriteArrayList所有的修改线程会对底层数组进行复制，这种操作是低效的。

LinkedList

2016-08-11

LinkedList是java里双链表的实现，LinkedList实现了list接口和Deque接口，可以存放任何值。同时LinkedList也是是线程不安全的。

LinkedList如何进行线程安全的操作。

需要使用Collections方法进行包装一下，包装成线程安全的类，同时如果要使用队列java.util.concurrent包中提供了两个线程安全的类LinkedBlockingDeque，和LinkedBlockingQueue可以直接那来用，不需要再做线程安全的同步，同时性能也比LinkedList同步性能更高。

LinkedList 类的实现

里面最小的存储单元，Node的实现

get方法

获取list中的元素的方法，可以看出linked元素是通过下标的移动来获取元素的，这和hashMap通过key的hash算法获取值时间复杂度相比较，list的时间复杂度是O（n）,而hashMap是O（1）。

put方法

LinkedList的插入总是在表头进行插入。

remove方法

链表的删除比较简单，不需要像数组那样进行删除数据移动n-1个位置，只需要交换指针指向，把移除的元素的指针指向null就可以完成删除，所以在linkedList中间位置删除元素比ArrayList速度更快。
