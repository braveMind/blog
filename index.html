    redis（挖坑）

    2016-05-08

    掌握

    1. Redis支持五种数据类型。
    2. Redis部分命令只能对特定的数据进行执行（Append只能针对字符串进行操作，HSET 只能对哈希表进行操作）而DEL，TYPE,EXPIRE可以对所有数据进行操作。
    3. Redis的数据库如何实现存储不同的数据类型的。
    4. 发布订阅，脚本，事务等特性。
    5. Redis如何处理客户端的请求。
    6. Redis位操作，排序，复制，Sentinel,集群。
    7. Redis单机特性，事务、持久化、Lua脚本、排序、二进制位操作。

    数据结构

    1. Redis是由键值对组成的，其中数据库的键值必须是字符串
    2. Redis的值value，可以是字符串，列表对象（）list object）,哈希对象（hash object）,集合对象（set object）,有序集合对象（sorted set object）这五种。
    3. Redis SDS 磁盘空间分配两种策略如果小于1M,那么分配free为字符串本身大小，如果大于1M那么free为1M，通过预分配策略可以减少内存分配的次数。

    中介者模式（行为模式四）

    2016-05-07

    中介者模式把类之间复杂的通信关系或复杂的关系引用，交由一个第三方的中介者类来处理，类之间的通信通过这个中介来维护，便于系统扩展。

    类的结构图

    分析

    中介模式的组成：

    1. 抽象的中介（Mediator）,定义中介处理消息的具体操作。
    2. 抽象的中介使用者类，也叫同事类（Person）利用中介来发送消息。
    3. 具体中介（ConcreteMediator）,具体的消息处理者 。
    4. 具体的的中介使用者（PersonA,PersonB），利用中介来处理具体业务。

    优点

    把系统间的通信交给抽象的第三方中介，解耦了系统，便于系统的维护和扩展。

    缺点

    在具体中介者类中包含了大量中介使用者之间的交互细节，可能会导致具体中介者类非常复杂，使得系统难以维护。

    迭代器模式（行为模式三）

    2016-05-05

    迭代器模式面对的是容器，为容器提供一系列的遍历，删除操作，作为客户并不直接操作容器，迭代器类为容器提供统一接口,遍历容器中的数据。

    类图

    分析

    迭代器的模式的组成：

    1. 抽象遍历接口，顶层Iterator提供统一容器遍历接口。
    2. 抽象容器（容器中不包含任何数据结构），定义容器需要的迭代器。
    3. 具体的迭代器 遍历具体的容器（内部类）。
    4. 具体容器定义具体的数据结构，对客户端提供一个遍历接口。

    优点

    迭代器模式把对象的遍历，和对象存放的容器解耦了，对容器的扩展很方便，对外提供了统一的遍接口，在JDK中应用广泛如Iterator接口，适用于遍历容器。

    缺点

    系统每增加一个容器，就需要增加一个迭代器类，增加了系统的复杂性。
